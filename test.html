Chapter 1: Introduction to Computer Systems

1.1 What is a Computer System?

A computer system is a collection of hardware and software components that work together to perform tasks. It accepts data (input), processes it, and produces information (output).

Key Components:

  • Hardware: The physical parts of the computer, such as the CPU, memory, and storage devices.
  • Software: The set of instructions that tell the hardware what to do, including operating systems and applications.
  • Data: Raw facts and figures that are processed by the computer.
  • Users: People who interact with the computer system.

1.2 Hardware Components

Understanding the core hardware components is crucial for system functionality.

CPU (Central Processing Unit):

The "brain" of the computer, responsible for executing instructions. Key characteristics include clock speed (GHz) and number of cores.

Image showing a typical CPU chip.

Memory (RAM - Random Access Memory):

Volatile storage used to hold data and instructions that the CPU is currently using. Faster than storage but loses data when power is off.

  • DRAM (Dynamic RAM): Commonly used type of RAM.
  • SRAM (Static RAM): Faster and more expensive than DRAM, used in CPU caches.

Storage Devices:

Non-volatile storage used to store data permanently.

  • HDD (Hard Disk Drive): Mechanical storage device using spinning platters.
  • SSD (Solid State Drive): Faster and more reliable than HDD, using flash memory.

Input/Output Devices:

Devices that allow the computer to interact with the outside world.

  • Input: Keyboard, mouse, scanner
  • Output: Monitor, printer, speakers

1.3 Software Components

Software provides the instructions for the hardware to execute.

Operating System (OS):

Manages hardware resources and provides a platform for applications to run. Examples include Windows, macOS, and Linux.

Application Software:

Programs designed to perform specific tasks, such as word processing, web browsing, or gaming.

System Software:

Utilities that help manage and maintain the computer system, such as antivirus software and disk defragmenters.

1.4 Number Systems (Binary, Decimal, Hexadecimal)

Computers use binary (base-2) to represent data. Understanding different number systems is essential for low-level programming and data representation.

Binary (Base-2):

Uses only two digits: 0 and 1.

Decimal (Base-10):

The number system we use in everyday life, using digits 0-9.

Hexadecimal (Base-16):

Uses digits 0-9 and letters A-F (A=10, B=11, C=12, D=13, E=14, F=15). Commonly used for representing memory addresses and colors.

Conversion Examples:

  • Decimal 10 = Binary 1010
  • Decimal 15 = Hexadecimal F

Chapter 2: Operating Systems

2.1 What is an Operating System?

An operating system (OS) is system software that manages computer hardware and software resources and provides common services for computer programs.

Key Functions:

  • Resource Management: Allocating resources like CPU time, memory, and I/O devices to different programs.
  • Process Management: Creating, scheduling, and terminating processes.
  • Memory Management: Managing memory allocation and deallocation.
  • File System Management: Organizing and managing files and directories.
  • User Interface: Providing a way for users to interact with the computer.

2.2 Types of Operating Systems

Different OS types cater to various needs.

Batch Operating System:

Processes jobs in batches without user interaction. Efficient for repetitive tasks.

Time-Sharing Operating System:

Allows multiple users to share the computer's resources simultaneously, providing interactive sessions.

Real-Time Operating System (RTOS):

Designed for applications that require immediate responses, such as industrial control systems and medical devices.

Distributed Operating System:

Manages a network of computers as a single system.

2.3 Process Management

Managing processes efficiently is critical for OS performance.

Process States:

  • New: Process is being created.
  • Ready: Process is waiting to be assigned to the CPU.
  • Running: Process is currently being executed by the CPU.
  • Waiting: Process is waiting for an event to occur (e.g., I/O completion).
  • Terminated: Process has finished execution.

Process Scheduling Algorithms:

Algorithms used to determine which process to run next.

  • First-Come, First-Served (FCFS): Processes are executed in the order they arrive.
  • Shortest Job First (SJF): Processes with the shortest execution time are executed first.
  • Priority Scheduling: Processes are assigned priorities, and higher priority processes are executed first.
  • Round Robin: Each process is given a fixed time slice, and processes are executed in a circular fashion.

2.4 Memory Management

Efficiently managing memory is vital for preventing system slowdowns.

Virtual Memory:

Technique that allows a computer to use more memory than is physically available by swapping portions of memory to disk.

Paging:

Dividing memory into fixed-size blocks called pages.

Segmentation:

Dividing memory into logical units called segments.

Diagram illustrating virtual memory concept.

2.5 File Systems

How the OS organizes and manages files.

File System Operations:

  • Create: Create a new file.
  • Read: Read data from a file.
  • Write: Write data to a file.
  • Delete: Delete a file.
  • Open: Open a file for reading or writing.
  • Close: Close a file.

Common File Systems:

  • FAT32: Older file system commonly used in USB drives.
  • NTFS: More advanced file system used in Windows.
  • ext4: Common file system used in Linux.

Chapter 3: Computer Networks

3.1 Introduction to Computer Networks

A computer network is a collection of interconnected computing devices that can communicate with each other and share resources.

Key Benefits:

  • Resource Sharing: Sharing printers, files, and internet connections.
  • Communication: Enabling communication through email, instant messaging, and video conferencing.
  • Data Backup and Recovery: Centralized data storage and backup.
  • Increased Efficiency: Streamlining workflows and improving productivity.

3.2 Network Topologies

The physical or logical arrangement of a network.

Bus Topology:

All devices are connected to a single cable (the bus). Simple but vulnerable to cable breaks.

Illustration of a Bus Topology.

Star Topology:

All devices are connected to a central hub or switch. More robust than bus topology.

Ring Topology:

Devices are connected in a closed loop. Data travels in one direction.

Mesh Topology:

Each device is connected to multiple other devices. Highly redundant but expensive.

3.3 Network Protocols

A set of rules that govern communication between devices on a network.

TCP/IP (Transmission Control Protocol/Internet Protocol):

The foundation of the internet. Defines how data is transmitted over a network.

  • TCP: Provides reliable, connection-oriented communication.
  • IP: Provides addressing and routing of data packets.

HTTP (Hypertext Transfer Protocol):

Used for transferring web pages and other web resources.

FTP (File Transfer Protocol):

Used for transferring files between computers.

SMTP (Simple Mail Transfer Protocol):

Used for sending email.

3.4 Network Devices

Hardware components used to build a network.

Hub:

Simple device that broadcasts data to all connected devices.

Switch:

More intelligent device that forwards data only to the intended recipient.

Router:

Connects different networks together and routes data between them.

Modem:

Converts digital signals to analog signals and vice versa, allowing computers to communicate over telephone lines.

3.5 Network Types

Categorizing networks based on their scope and size.

LAN (Local Area Network):

Connects devices within a limited area, such as a home, office, or school.

WAN (Wide Area Network):

Connects devices over a large geographical area, such as a city, country, or the world.

MAN (Metropolitan Area Network):

Connects devices within a metropolitan area, such as a city or a large campus.
